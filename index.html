Semantic HTML Elements Reference Table

Opting for semantic HTML elements can be a valuable strategy to maintain site accessibility, optimize for SEO (Search Engine Optimization), and enhance code comprehensibility. When utilizing semantic HTML elements, your choices are driven by their meanings. However, elements like <div> and <span> aren't considered semantic because they lack contextual information about their enclosed content. Recognizing the significance of this topic, I chose to create a Semantic HTML Elements Reference Table as a practical exercise presented by Codecademy in the Challenge Project: Build Your Own Cheat Sheet.

Tag Name Description
<header> Header Represents a part of a page which is meant to be introductory. It can include headings tags, a logo, a seach bar, and navigation elements.
<nav> Navigation Links It is used to define a block of navigation links such as menus and table of contents.
<main> Main It is a structural element and is used to encapsulate the dominant content within the page.
<footer> Footer It is a structural element and has encapsulated the content at the bottom of webpage (e.g. contact information, copyright information, terms of use, etc.)
<section> Section Defines elements in a document, such as chapters, headings, or any other area of the document with the same theme.
<article> Article Represents a part of a page which is self-contained and could be published elsewhere. Common uses include blog posts or magazine articles.
<aside> Aside It's used to mark additional information that can enhance another element but it is not required in order to undersatnd the main content.
<figure> Figure It is an element to encapsulate media such as an image, illustration, diagram, code snippet, etc., which is referenced in the main flow of the document.
<figurecaption> Figure Caption It is an element used to describe the media in the <figurecaption> tag.
<audio> Audio It is used to embed audio content in a document.
<video> Video It is used to embed video content in a document.
<embed> Embed It can embed any media from an external source content. It's a self close tag.
